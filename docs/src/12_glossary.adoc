[[section-glossary]]
== Glossary

[cols="e,2e", options="header"]
|===
|Term |Definition

|YOVI
|Name of the system developed in this project. A web-based platform for playing Game Y, supporting human players and external bots.

|Game Y
|An abstract strategy board game in which players aim to connect three different sides of the board.

|Classic Game Y
|The standard version of Game Y defined by the original rules, without additional variants or optional mechanics.

|YEN Notation
|A JSON-based notation used to represent the state of a Game Y match, including board size, current turn, players, and board layout.

|Game State
|A complete representation of a Game Y match at a specific point in time, encoded using YEN notation.

|TypeScript
|A statically typed programming language that extends JavaScript and is used to implement the web application.

|Rust
|A systems programming language focused on safety and performance, used to implement the game engine module.

|API
|A public HTTP-based interface exposed by the web application that allows external clients and bots to interact with the system.

|Bot
|An external program that interacts with the system exclusively through the public API to retrieve information and play games automatically.

|Player-versus-Machine
|A game mode in which a human player competes against a computer-controlled opponent.

|JSON
|A lightweight data-interchange format used for communication between the web application and the Rust game engine.

|HTTP
|The application-layer protocol used for communication between clients, bots, and system services.

|===
